1. Onyx (Danswer) là gì?

Giới thiệu

Onyx (trước đây là Danswer) là một nền tảng trí tuệ nhân tạo thế hệ mới (GenAI) phục vụ tìm kiếm thông tin nội bộ và tích hợp LLMs dành riêng cho doanh nghiệp. Khác với các ứng dụng chatbot thông thường, Onyx đóng vai trò như một hệ thống AI nội bộ tập trung của toàn bộ tổ chức, kết hợp giữa việc hỏi đáp với LLMs, tra cứu tri thức chéo từ hàng chục nguồn dữ liệu, và thực thi mã lệnh tự động.

Hệ thống hỗ trợ cơ chế tự động đồng bộ quyền truy cập (Auto Sync Permissions) từ các nguồn gốc như Google Drive, Gmail, Confluence, Slack, Jira... đảm bảo nhân viên chỉ có thể tìm thấy dữ liệu mà họ được phép xem.

2. Các tính năng cốt lõi vượt trội

Tính năng lõi
  • Code Interpreter (Sandbox Python): Cho phép mô hình tự động viết và thực thi mã Python trong môi trường ảo biệt lập (sandbox). Dùng để xử lý các phép toán phức tạp, phân tích dữ liệu từ file upload và trực quan hóa sơ đồ.
  • 40+ Persistent Connectors: Khung kết nối bền bỉ tự động đồng bộ định kỳ dữ liệu từ Drive, Notion, GitHub, Confluence, Jira, HubSpot, Teams... giúp tri thức nội bộ luôn được cập nhật.
  • Phân quyền Curator & Global Curator: Thiết lập lớp quản trị trung gian ở cấp phòng ban. Curator có thể quản lý dữ liệu, connectors và trợ lý riêng của nhóm mình mà không cần can thiệp quyền Admin hệ thống.
  • Deep Research: Chế độ nghiên cứu sâu tự động thực hiện chuỗi hành động tìm kiếm, đọc hiểu, suy luận đa tầng để tổng hợp thông tin phức tạp (tốn khoảng 10 lần tài nguyên so với chat thường).

3. Kiến trúc hệ thống đa tầng (Architecture)

Technical Details

Kiến trúc của Onyx được đóng gói dưới dạng tập hợp các container Docker độc lập, phân rã nhiệm vụ rõ ràng:

┌────────────────────────────────────────────────────────┐ │ APPLICATION LAYER │ │ - Frontend: Next.js Web Client │ │ - Backend: FastAPI (Python) Web Server │ │ - Background Workers: Celery/Celery Beat (Async Jobs) │ └──────────────────────────┬─────────────────────────────┘ ▼ ┌────────────────────────────────────────────────────────┐ │ DATA LAYER │ │ - Postgres: Lưu trữ dữ liệu ứng dụng & trạng thái │ │ - Vespa: Lưu trữ vector & Keyword (Hybrid Search) │ │ - Redis: Cache lưu trữ đệm tốc độ cao │ │ - MinIO: Lưu trữ đối tượng (S3 compatible) │ └────────────────────────────────────────────────────────┘

Frontend viết bằng Next.js giao tiếp trực tiếp với Core Backend FastAPI (Python). Công cụ tìm kiếm hybrid kết hợp giữa từ khóa (lexical) và vector (semantic) được vận hành trên hạ tầng Vespa engine. File tài liệu nạp lên được lưu trữ tập trung tại MinIO (có thể cấu hình trỏ sang AWS S3).

4. Pháp lý & Thương mại phần mềm (Compliance)

Onyx phân tách rõ ràng giữa phiên bản Self-Hosted Open-Source (miễn phí) và bản thương mại Enterprise. License kích hoạt được quản lý bằng Stripe subscription và xác thực cục bộ (locally validated) bằng chữ ký số, hỗ trợ tốt cả trong môi trường mạng nội bộ cô lập (air-gapped environments).

Khi triển khai thương mại ra toàn cầu, Onyx phải xử lý 4 nhóm compliance pháp lý:

  • Thuế dịch vụ số: Khai báo thuế VAT chéo biên giới (qua cơ chế EU OSS).
  • Export controls (Kiểm soát xuất khẩu): Tuân thủ quy định EAR của Mỹ đối với các công nghệ mã hóa và phần mềm.
  • Dữ liệu cá nhân (GDPR): Đóng vai trò là Processor hoặc Controller khi xử lý dữ liệu người dùng EU, áp dụng điều khoản truyền dữ liệu SCCs.
