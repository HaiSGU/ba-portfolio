1. Secure Staging & Row Deduplication

Phase 1

To safeguard the raw imported database, I created a safe staging environment `layoffs_staging`. To execute 100% accurate deduplication without database key collations, I mapped rows using a `ROW_NUMBER() OVER` partition logic across all columns.

Because standard database engines do not support direct `DELETE` operations on complex CTEs, I created a secondary staging schema `layoffs_staging2` adding a helper column `row_num`, inserted the row calculations, deleted duplicate entries, and then dropped the helper: 

-- Safe Deduplication Architecture
CREATE TABLE `layoffs_staging2` (
  `company` text, `location` text, `industry` text, `total_laid_off` int,
  `percentage_laid_off` text, `date` text, `stage` text, `country` text,
  `funds_raised_millions` int, `row_num` INT
);

INSERT INTO layoffs_staging2
SELECT *, ROW_NUMBER() OVER (
  PARTITION BY company, location, industry, total_laid_off, percentage_laid_off, `date`, stage, country, funds_raised_millions
) AS row_num 
FROM layoffs_staging;

-- Hard delete duplicates
DELETE FROM layoffs_staging2 WHERE row_num >= 2;
ALTER TABLE layoffs_staging2 DROP COLUMN row_num;

2. Categorical Standardization & Type Casting

Phase 2

Raw text datasets often suffer from split labels (e.g. having `Crypto Currency`, `CryptoCurrency`, and `Crypto` representing the same sector) and trailing punctuation. I normalized these records, aligned blanks to standard database NULLs, and imputed empty industries by joining the staging table against itself based on corporate matches: 

-- Standardizing labels & Imputing nulls via self-join
UPDATE layoffs_staging2 SET industry = 'Crypto' WHERE industry IN ('Crypto Currency', 'CryptoCurrency');
UPDATE layoffs_staging2 SET country = TRIM(TRAILING '.' FROM country);

UPDATE layoffs_staging2 t1
JOIN layoffs_staging2 t2 ON t1.company = t2.company
SET t1.industry = t2.industry
WHERE t1.industry IS NULL AND t2.industry IS NOT NULL;

Chronological Type Casting: String date variables block sorting or interval operations. I parsed string dates using `STR_TO_DATE` and modified the table column format to standard date values: 

-- Parse date string and alter column structure to proper Date type
UPDATE layoffs_staging2 SET `date` = STR_TO_DATE(`date`, '%m/%d/%Y');
ALTER TABLE layoffs_staging2 MODIFY COLUMN `date` DATE;

3. Advanced CTE exploratory Analytics

Phase 3

With the database clean and indexed, I executed advanced analytical queries using nested **Common Table Expressions (CTEs)** and **Window Functions** to uncover macro layoffs behaviors:

Query A: Rolling Monthly Layoffs Total

Reveals the overall chronological slope of global layoffs by accumulating month-over-month sums.

WITH DATE_CTE AS (
    SELECT SUBSTRING(date,1,7) as dates, SUM(total_laid_off) AS total_laid_off
    FROM layoffs_staging2
    GROUP BY dates ORDER BY dates ASC
)
SELECT dates, SUM(total_laid_off) OVER (ORDER BY dates ASC) as rolling_total
FROM DATE_CTE ORDER BY dates ASC;

Query B: Top 3 Corporate Layoffs Ranks Per Year

Utilizes double CTE layers and the `DENSE_RANK()` partition window to rank top laid-off companies within their respective calendar years.

WITH Company_Year AS (
    SELECT company, YEAR(date) AS years, SUM(total_laid_off) AS total_laid_off
    FROM layoffs_staging2
    GROUP BY company, YEAR(date)
), Company_Year_Rank AS (
    SELECT company, years, total_laid_off, 
           DENSE_RANK() OVER (PARTITION BY years ORDER BY total_laid_off DESC) AS ranking
    FROM Company_Year
)
SELECT company, years, total_laid_off, ranking
FROM Company_Year_Rank
WHERE ranking <= 3 AND years IS NOT NULL
ORDER BY years ASC, total_laid_off DESC;

4. Key Macro Insights

  • Macro Volume surge: The timeline reveals a significant layoff surge starting in Q3 2022, peaking in Q1 2023, primarily concentrated in tech, retail, and consumer sectors.
  • Geographical Footprint: The United States tech clusters carried over 75% of overall laid-off volume, dominated by hyper-scale enterprise downsizings.
  • Funding Volatility: Companies raising massive venture capital (e.g. Quibi raising $2B+) collapsed completely, proving that high capital cushions do not substitute for product-market fit or cashflow sustainability.
